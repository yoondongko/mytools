# 내 포지션 신호등 — stock-signal.html

## 최근 수정 이력

### 2026-06-14 (2차 업데이트)

#### 새로 추가된 기능
- **상단 요약 그리드** — 페이지 최상단에 전체 종목 요약 카드 표시. 종목명·매입단가·현재가·대비%·신호를 한눈에 확인. 카드를 클릭하면 해당 상세 카드로 스크롤. 3개면 1행×3열, 4개 이상은 3열 고정으로 줄바꿈. 관망 탭 제거, 해당 구간은 버티기로 흡수.
- **가격 추이 차트** — 각 종목 카드에 Canvas 기반 라인 차트 추가.
  - **일별** 탭: 당일(또는 최근 24시간) 누적 가격 데이터 표시
  - **주별** 탭: 최근 30일 중 각 날짜 마지막 가격 표시
  - 매입평균단가가 금색 점선으로 차트에 함께 표시됨
  - 현재가가 매입단가 이상이면 빨간 선, 미만이면 파란 선
  - 가격 데이터는 60초 자동 조회 또는 수동 입력 시마다 자동 누적 (최대 10,000건 보관)
- **종목 이름 수정** — 카드의 종목명을 클릭하면 직접 편집 가능. Enter로 저장, Esc로 취소.
- **메모 칸** — 각 카드 하단에 자유 메모 입력 영역 추가. 실시간 저장.
- **배경 밝게** — 전체 UI를 연한 배경의 라이트 톤으로 변경.

#### 레이아웃 수정
- **매수/매도 창 오버플로우 수정** — 기존 나란히 배치(2열 grid) → 세로 2줄 배치(flex column)로 변경. 각 행은 `34px / 1fr / 1fr / 52px` 4열 grid로 깔끔하게 정렬됨.

### 2026-06-14 (1차 업데이트)

#### 수정된 문제
- **한글 인코딩 깨짐 복구** — 파일이 UTF-8 BOM으로 저장되면서 한글 바이트가 `?`로 교체되어 전체 텍스트가 깨진 상태였음. git 원본(`899e5ce`)을 참고해 복원.
- **비밀번호 게이트 제거** — `auth.js` 로드 및 인라인 IIFE(passGate 관련 코드)가 존재하지 않는 HTML 요소를 참조해 JS 전체가 실행 불가 상태였음. 관련 코드 전체 삭제.
- **UTF-8 BOM 제거** — 인코딩 이상의 원인이었던 BOM(`EF BB BF`) 제거.

## 자동 새로고침(60초) 작동 여부 안내

### 현재 코드 동작
- 60초마다 자동으로 네이버 시세 데이터를 fetch하도록 구현되어 있음.
- 하지만 실제 작동 여부는 실행 환경에 따라 다름.

### 문제 가능성
- 네이버 시세 API가 GitHub Pages 같은 외부 도메인에서의 요청을 CORS 정책으로 차단할 수 있음.
- 차단되면 화면에 "자동조회 실패 — 현재가/전일가를 직접 입력해주세요"라는 메시지가 표시됨.
- 차단 시 카드의 현재가/전일가 입력칸에 직접 값을 입력하면 신호가 즉시 갱신됨.

### 확인 방법
1. 브라우저 개발자도구 열기 (F12)
2. Console 탭 확인
3. "blocked by CORS policy" 관련 에러 메시지가 있는지 확인

### 막혔을 경우 해결 방법

#### 방법 1: GitHub Actions로 주기적 갱신
- GitHub Actions가 주기적으로 서버 쪽에서 시세를 가져옴
- 가져온 데이터를 JSON 파일로 저장
- 웹페이지는 이 JSON 파일만 읽도록 수정
- 장점: CORS 문제 없음
- 단점: GitHub Actions 최소 실행 주기에 한계 있음

#### 방법 2: CORS 프록시 서버 사용
- Cloudflare Workers 같은 무료 서비스로 프록시 서버 구축
- 프록시를 거쳐서 네이버 API 호출
